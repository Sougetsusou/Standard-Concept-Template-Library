# Template Code Edit Plan

---

# Overview

Three-pass optimization strategy. Each pass is independent and safe to stop after.

- **Pass 1** — bug fixes in place, zero signature changes, zero caller impact
- **Pass 2** — rename incompatible duplicates, fix flat-param signatures, regenerate pkl files
- **Pass 3** — consolidate identical duplicates into a shared module, extract base classes

---

# Working Directory Strategy

## Where to make edits

**Work directly in `code/<Category>/concept_template.py`** — not in `filtered_template/`.

Reasons:
- The visualization pipeline (`visualize.py`, `visualize_knowledge.py`) imports directly from `code/<Category>/concept_template.py` and loads `code/<Category>/conceptualization.pkl`. Edits there are immediately testable.
- Each fix can be verified per-category by running `visualize.py` after editing, confirming geometry is unchanged for correct classes and fixed for buggy ones.
- The same bug appearing in multiple categories (e.g. `self.mesh` loop overwrite) is fixed once per category file — 39 files total, but each fix is small and scoped.

## How `filtered_template/` fits in

- `filtered_template/` was generated by `scripts/extract_and_classify_templates.py` and is used for review only.
- After all passes are complete, re-run the extraction script to regenerate `filtered_template/` as a clean snapshot of the optimized code. Do not edit `filtered_template/` directly.

## Testing workflow per category

For each category edited in Pass 1/2:
1. Edit `code/<Category>/concept_template.py`
2. Run `python visualize.py` in `code/<Category>/` to confirm geometry renders correctly
3. If Pass 2 involves a class rename, also update the `conceptualization.pkl` reference and re-verify

## Keeping originals

`code_original/` is a one-time physical archive of all 39 categories created before Pass 1 begins. It is never edited. Use it as a reference or fallback if needed.

All actual work happens in `code/` with git tracking. Each pass is a set of commits. If anything breaks, `git diff` and `git checkout` are the recovery tools. The original is always recoverable from either `code_original/` or git history.

# Pass 1: In-place Bug Fixes (no signature changes)

**Goal:** Make every class correct and crash-free without touching any caller or pkl file.
**Constraint:** No parameter names, no class names, no import structure changes.

## Step 1.1 — Run duplicate-body audit script
Before touching any code, run a script that hashes all class bodies across all template files and outputs:
- Which classes are byte-for-byte identical (safe to ignore until Pass 3)
- Which classes share a name but differ in body (need attention in Pass 2)

This gives a ground-truth map so we don't accidentally "fix" a class that is about to be deleted.

## Step 1.6 — Major logic restructuring (biggest readability wins)

These are the changes that most dramatically reduce line count and improve readability. Each is a logic rewrite with no signature change.

### 1.6a Bilateral symmetry — replace 4-block unroll with direction loop

**Impact:** Eliminates 60–80% of code in affected classes. The single most impactful change in the library.

**Pattern:** Front-left / behind-left / front-right / behind-right (or top/bottom, left/right) written as 4 separate copy-pasted blocks where only a sign or index flips.

**Fix:** Replace with a loop over a direction table.

```python
# Before: 4 × 20 lines = ~80 lines
# front-left block ...
# behind-left block ...
# front-right block ...
# behind-right block ...

# After: ~10 lines
directions = [
    ( 1,  1),   # front-left
    ( 1, -1),   # behind-left
    (-1,  1),   # front-right
    (-1, -1),   # behind-right
]
for x_sign, y_sign in directions:
    pos = [x_sign * offset[0], y_sign * offset[1], 0]
    tmp_mesh = Cuboid(..., position=pos)
    vertices_list.append(tmp_mesh.vertices)
    faces_list.append(tmp_mesh.faces + total_num_vertices)
    total_num_vertices += len(tmp_mesh.vertices)
```

**Affected:**
- `gripper_templates.py` (`Cusp_Gripper` — 4 bilateral segments)
- `handle_templates.py` (all 4 Pliers classes — front/behind × left/right unroll; all 4 Door handle classes — direction-settings loop)
- `cover_templates.py` (`Fourfold_Cover` — front/back/left/right; `Holed_Cuboidal_Cover` — 4 side panels)
- `doorframe_templates.py` (`Standard_Doorframe` — sub1/sub2 layer pair)
- `back_templates.py` (`Ladder_back`, `Splat_back`, `Latice_back`, `Slat_back` — vertical post pair)
- `baffle_templates.py` (`Rectangular_Baffle`, `Curved_Baffle` — left/right pair)
- `body_templates.py` (`Standard_Body` shaft pair, `Cambered_Body` half-cylinder pair)
- `plug_templates.py` (`Standard_Plug` — 3-pin unroll)
- `board_templates.py` (`Regular_partition` — left/right panel pair)
- `base_templates.py` (`UShapedXZ_Base`, `UShapedYZ_Base`)

### 1.6b Existence-flag dispatch — replace N `if` blocks with a data-driven loop

**Impact:** Reduces N near-identical conditional blocks to one loop. Makes adding/removing panels trivial.

**Pattern:** `if has_X[0] == 1:` blocks that are structurally identical except for which geometry they build and which axis/sign they use.

**Fix:** Build a list of `(flag, params)` tuples and loop over it.

```python
# Before: 4 separate if blocks, ~60 lines
if has_front[0] == 1:
    mesh = Cuboid(...front params...)
    ...
if has_back[0] == 1:
    mesh = Cuboid(...back params...)
    ...

# After: ~15 lines
panels = [
    (has_front[0],  front_size,  [0, 0,  depth/2]),
    (has_back[0],   back_size,   [0, 0, -depth/2]),
    (has_left[0],   left_size,   [-width/2, 0, 0]),
    (has_right[0],  right_size,  [ width/2, 0, 0]),
]
for exists, size, pos in panels:
    if not exists:
        continue
    tmp_mesh = Cuboid(*size, position=pos)
    vertices_list.append(tmp_mesh.vertices)
    faces_list.append(tmp_mesh.faces + total_num_vertices)
    total_num_vertices += len(tmp_mesh.vertices)
```

**Affected:**
- `cover_templates.py` (`Fourfold_Cover`, `Holed_Cuboidal_Cover`)
- `board_templates.py` (`Regular_partition`)
- `support_templates.py` (`Trifold_Support`)
- `door_templates.py` (`Standard_Door`)
- `switch_templates.py` (`RotaryX_Switch`)
- `doorframe_templates.py` (`Standard_Doorframe` — sub1/sub2 existence pair)
- `bracket_templates.py` (`Semi_Ring_Bracket` — top/bottom endpoint pair)

### 1.6c `locals()` hack — collect numbered params into a list at top of `__init__`

**Impact:** Eliminates all `locals()` calls. No signature change needed. Turns opaque dynamic lookups into readable indexed access.

**Fix:** At the top of `__init__`, before any geometry construction, collect the numbered params:

```python
# Before (scattered through the method):
size = locals()['burner_%d_size' % (i+1)]
offset = locals()['burner_%d_central_offset' % (i+1)]

# After (collected once at top):
burner_sizes   = [burner_1_size,   burner_2_size,   ..., burner_6_size]
burner_offsets = [burner_1_central_offset, ..., burner_6_central_offset]
# then in loop:
size   = burner_sizes[i]
offset = burner_offsets[i]
```

**Affected:** `switch_templates.py` (`Round_Switch`), `spout_templates.py` (`Straight_Spout`), `nozzle_templates.py` (`Press_Nozzle`), `cover_templates.py` (`Standard_Cover`), `body_templates.py` (`Mutiple_Layer_Body`, `Multilevel_Body`), `button_templates.py` (`Controller_With_Button`), `burner_templates.py` (`Top_With_Burner`)

### 1.6d Multi-step pre-transform chains — compose transforms and apply once

**Impact:** Replaces 2–4 sequential opaque vertex mutations with a single named, documented transform. Makes the geometric intent verifiable.

**Pattern:** 2–4 sequential `apply_transformation(mesh.vertices, ...)` calls building up a composed transform step by step, with no intermediate variable names explaining the intent.

**Fix:** Name each intermediate step clearly, or compose the rotation/translation mathematically and apply once. For pivot rotations, add a comment explaining the pivot point and axis.

```python
# Before:
self.mesh.vertices = apply_transformation(self.mesh.vertices, [0,0,0], rotation_1)
self.mesh.vertices = apply_transformation(self.mesh.vertices, offset, rotation_2)
self.mesh.vertices = apply_transformation(self.mesh.vertices, pivot, rotation_3)

# After:
# Step 1: align to local axis
tmp_mesh.vertices = apply_transformation(tmp_mesh.vertices, [0,0,0], rotation_1)
# Step 2: translate to arm position
tmp_mesh.vertices = apply_transformation(tmp_mesh.vertices, offset, rotation_2)
# Step 3: pivot about base joint
tmp_mesh.vertices = apply_transformation(tmp_mesh.vertices, pivot, rotation_3)
```

**Affected:** `leg_templates.py` (`Star_leg` — 4 chained Rodrigues transforms), `switch_templates.py` (`RegularY/X/Z_Switch`, `Cuboidal_Switch`, `Faucet_Lever_Switch`), `spout_templates.py` (`Trifold/ShowerRose/Quadfold_Spout`), `glasses_templates.py` (all 4 classes), `cap_templates.py` (`SquareEnded_Cap`, `RoundEnded_Cap`)

### 1.6e Axis-variant class families — merge into single class with axis parameter

**Impact:** Replaces 2–3 near-identical classes with one. Eliminates the largest blocks of structural duplication that aren't byte-for-byte identical.

**Candidates:**

| Family | Classes | Difference | Merged parameter |
|---|---|---|---|
| Rotary Switch | `RotaryX_Switch`, `RotaryY_Switch`, `RotaryZ_Switch` | axis of rotation only | `axis='X'/'Y'/'Z'` |
| U-shaped Base | `UShapedXZ_Base`, `UShapedYZ_Base` | plane only | `plane='XZ'/'YZ'` |
| Flip Switch | `FlipX_Switch`, `FlipY_Switch` | rotation axis only | `axis='X'/'Y'` |
| Regular/Cylindrical sublayer | `Regular_sublayer`, `Cylindrical_sublayer` | mesh primitive only | `shape='cuboid'/'cylinder'` |

**Fix:** Write one merged class, keep the old names as thin aliases for backward compatibility:

```python
class RotaryX_Switch(Rotary_Switch):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, axis='X', **kwargs)
```

This keeps pkl compatibility (class name unchanged) while eliminating the duplicated logic.

## Step 2.3 — Fix `locals()` hack signatures

def __init__(self, button_1_size, button_1_offset, ..., button_10_size, button_10_offset, num_buttons, ...):
